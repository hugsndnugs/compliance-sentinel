    Troubleshooting¶

    Common issues and solutions for the Discord Legal Compliance Generator bot.

    Bot Connection Issues¶

    Bot Doesn't Connect¶

    Symptoms: - Bot doesn't appear online in Discord - No startup messages in console - Connection errors in logs

    Solutions:

    1. Check Bot Token: 

      # Verify .env file exists and has correct token
cat .env
      Ensure DISCORD_BOT_TOKEN is set correctly.

    2. Verify Token Validity:

    3. Go to Discord Developer Portal
    4. Check if token is still valid

    5. Reset token if needed and update .env

    6. Check Internet Connection:

    7. Ensure your server has internet access
    8. Check firewall settings

    9. Verify Discord API is accessible

    10. Review Logs: 

      # Check for specific error messages
# Common errors:
# - "401 Unauthorized" - Invalid token
# - "403 Forbidden" - Missing permissions
# - Connection timeout - Network issues

    Bot Connects But Immediately Disconnects¶

    Symptoms: - Bot appears online briefly then goes offline - Repeated connection/disconnection cycles

    Solutions:

    1. Check Bot Intents:
    2. Verify intents are enabled in Discord Developer Portal

    3. Ensure message_content intent is enabled if using message commands

    4. Review Error Logs:

    5. Check for exception errors in console

    6. Look for permission-related errors

    7. Verify Python Version: 

      python --version
# Should be 3.10 or higher

    Command Issues¶

    Slash Commands Not Appearing¶

    Symptoms: - /generate command doesn't show in Discord - Commands not in autocomplete

    Solutions:

    1. Wait for Sync:
    2. Slash commands can take up to 1 hour to sync globally
    3. Restart Discord client

    4. Try in a server where bot was just added

    5. Check Bot Permissions:

    6. Verify bot has "Use Slash Commands" permission

    7. Ensure bot has "applications.commands" scope

    8. Force Command Sync: 

      # In bot.py, ensure setup_hook includes:
synced = await self.tree.sync()
logger.info(f"Synced {len(synced)} slash command(s)")

    9. Verify Command Registration:

    10. Check bot/commands/generate.py has @app_commands.command decorator
    11. Ensure cog is loaded in setup_hook()

    Commands Not Responding¶

    Symptoms: - Bot doesn't respond to /generate or !generate - No error messages

    Solutions:

    1. Check Bot Status:
    2. Verify bot is online (green status)

    3. Check bot is in the server/channel

    4. Verify Permissions:

    5. Bot needs "Send Messages" permission
    6. Check channel permissions

    7. Verify bot role has necessary permissions

    8. Check Command Prefix: 

      # Verify BOT_PREFIX in .env matches what you're using
# Default is "!"

    9. Review Bot Logs:

    10. Check for error messages
    11. Look for permission denied errors

    Form Modal Not Appearing¶

    Symptoms: - Clicking document type button doesn't show form - Modal doesn't open

    Solutions:

    1. Update Discord Client:
    2. Modals require recent Discord client version

    3. Update Discord to latest version

    4. Check Interaction Response:

    5. Ensure modal is sent before timeout

    6. Verify interaction.response.send_modal() is called

    7. Review Form Handler:

    8. Check form_handler.py for errors
    9. Verify modal is properly constructed

    Document Generation Issues¶

    Template Not Found Error¶

    Symptoms: - Error: "Template not found for {document_type}" - FileNotFoundError in logs

    Solutions:

    1. Verify Template Exists: 

      ls bot/templates/
# Should show all .md files

    2. Check Template Name:

    3. Template filename must match document type key
    4. Example: rules.md for document type "rules"

    5. Case-sensitive on some systems

    6. Verify Template Directory: 

      # Check TEMPLATES_DIR in config/config.py
# Should point to bot/templates/

    Variables Not Replacing¶

    Symptoms: - {variable_name} appears literally in generated document - Placeholders not filled

    Solutions:

    1. Check Variable Names:
    2. Variable names in template must match form data keys
    3. Case-sensitive matching

    4. Check for typos

    5. Verify Form Data: 

      # In form_handler.py, ensure form_data dictionary
# has keys matching template variables
form_data = {
    "server_name": server_name.value,  # Matches {server_name}
    # ...
}

    6. Check Template Encoding:

    7. Ensure template files are UTF-8 encoded
    8. Check for hidden characters

    Document Too Long¶

    Symptoms: - Error about message length - Document truncated

    Solutions:

    1. Automatic File Handling:
    2. Bot automatically sends long documents as files

    3. Check for .md file attachment

    4. Split Content:

    5. Break long documents into sections

    6. Use multiple messages if needed

    7. Template Optimization:

    8. Remove unnecessary content
    9. Use concise formatting

    Configuration Issues¶

    Environment Variables Not Loading¶

    Symptoms: - Bot token not found - Configuration errors

    Solutions:

    1. Verify .env File: 

      # Check file exists
ls -la .env

# Verify format (no spaces around =)
DISCORD_BOT_TOKEN=your_token_here
BOT_PREFIX=!

    2. Check python-dotenv: 

      pip install python-dotenv

    3. Verify Loading: 

      # In config.py, ensure:
from dotenv import load_dotenv
load_dotenv()

    Bot Prefix Not Working¶

    Symptoms: - !generate doesn't work - Wrong prefix expected

    Solutions:

    1. Check .env File: 

      BOT_PREFIX=!
# Or your preferred prefix

    2. Restart Bot:

    3. Configuration loads on startup

    4. Restart after changing prefix

    5. Verify Message Content Intent:

    6. Required for message commands
    7. Enable in Discord Developer Portal

    Permission Issues¶

    Missing Permissions Error¶

    Symptoms: - "Missing Permissions" error - Bot can't send messages

    Solutions:

    1. Check Bot Permissions:
    2. Server Settings Roles Bot Role

    3. Enable: Send Messages, Embed Links, Attach Files

    4. Channel Permissions:

    5. Right-click channel Edit Channel Permissions

    6. Ensure bot role has necessary permissions

    7. Verify Invite URL:

    8. Re-invite bot with correct permissions
    9. Use OAuth2 URL Generator in Developer Portal

    Performance Issues¶

    Bot Slow to Respond¶

    Symptoms: - Commands take long to process - Delayed responses

    Solutions:

    1. Check System Resources:
    2. Monitor CPU and memory usage

    3. Check for resource constraints

    4. Review Logging:

    5. Excessive logging can slow bot

    6. Adjust log level if needed

    7. Template Size:

    8. Large templates may slow generation
    9. Optimize template content

    High Memory Usage¶

    Symptoms: - Bot uses excessive memory - System slowdown

    Solutions:

    1. Check Active Forms:
    2. active_forms dictionary stores form data
    3. Forms timeout after 5 minutes

    4. Consider reducing timeout if needed

    5. Template Caching:

    6. Templates are loaded on-demand
    7. Consider caching for frequently used templates

    Development Issues¶

    Import Errors¶

    Symptoms: - ModuleNotFoundError - Import failures

    Solutions:

    1. Verify Installation: 

      pip install -r requirements.txt

    2. Check Python Path: 

      # Run from project root
python main.py

    3. Virtual Environment: 

      # Ensure virtual environment is activated
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate      # Windows

    Syntax Errors¶

    Symptoms: - Python syntax errors - Indentation errors

    Solutions:

    1. Check Python Version: 

      python --version
# Should be 3.10+

    2. Use Linter: 

      pip install flake8
flake8 bot/

    3. Verify Indentation:

    4. Use consistent indentation (4 spaces recommended)
    5. Check for mixed tabs/spaces

    Getting Help¶

    Before Asking for Help¶

    1. Check Logs:
    2. Review console output

    3. Look for error messages

    4. Verify Setup:

    5. Follow installation guide

    6. Check all prerequisites

    7. Search Issues:

    8. Check GitHub issues
    9. Search error messages

    Providing Information¶

    When reporting issues, include:

    • Error Messages: Full error text
    • Logs: Relevant log output
    • Steps to Reproduce: What you did
    • Expected Behavior: What should happen
    • Actual Behavior: What actually happened
    • Environment: Python version, OS, discord.py version

    Useful Commands¶

    # Check Python version
python --version

# Check installed packages
pip list

# Verify bot token format
echo $DISCORD_BOT_TOKEN  # Should not be empty

# Test template loading
python -c "from bot.handlers.document_generator import DocumentGenerator; d = DocumentGenerator(); print(d.load_template('rules'))"

    Common Error Messages¶

    "401 Unauthorized"¶

    • Invalid or expired bot token
    • Solution: Reset token in Developer Portal

    "403 Forbidden"¶

    • Missing bot permissions
    • Solution: Re-invite bot with correct permissions

    "Command not found"¶

    • Slash commands not synced
    • Solution: Wait for sync or restart bot

    "Template not found"¶

    • Missing template file
    • Solution: Verify template exists in bot/templates/

    "Missing required argument"¶

    • Incomplete command usage
    • Solution: Check command syntax in documentation
